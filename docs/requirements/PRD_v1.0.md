# ALIGNN晶体性质预测服务 - 需求文档 v1.0

> 目标用户：企业研发（电池/材料公司）
> 使用场景：单结构探索 + 批量筛选 并重
> 约束：公开服务，纯科研工具，无商业化计划

---

## 1. 产品概述

**产品名称**：ALIGNN 晶体性质预测平台

**一句话描述**：面向企业研发团队的材料性质预测工具，基于原子级线图神经网络(ALIGNN)，支持单结构和批量构型的快速性质预测，辅助电池材料研发中的掺杂筛选与优化。

**核心价值**：
- 快速：分钟级获取材料性质预测，替代昂贵的DFT计算
- 批量：一次生成并评估数百个掺杂构型
- 可视化：3D晶体结构 + 预测结果联动展示

## 3. 功能需求

### 3.1 核心功能矩阵

| 功能 | 优先级 | 描述 | 用户收益 |
|-----|-------|------|---------|
| **单结构预测** | P0 | 上传晶体文件，选择模型，立即获得性质预测 | 快速验证假设 |
| **批量预测** | P0 | 批量上传多个结构文件，后台异步处理，批量返回结果 | 高效筛选大量候选 |
| **自定义掺杂生成** | P1 | 用户指定宿主结构、掺杂元素、浓度梯度，自动生成构型 | 定制化研究 |
| **多数据库对比** | P1 | 同时使用JARVIS和MP模型预测，交叉验证结果可靠性 | 提升预测置信度 |
| **3D晶体可视化** | P1 | 上传即展示3D结构，高亮掺杂原子位置 | 直观理解结构 |
| **实验数据对比** | P2 | 输入实验数据，与预测值对比，计算偏差 | 验证模型准确性 |
| **文献数据关联** | P2 | 展示相关文献中的实验值和计算值 | 辅助结果解读 |
| **批量数据管理** | P2 | 管理历史预测记录，分组、标记、导出 | 组织研究数据 |

### 3.2 详细功能说明

#### F1: 单结构预测（P0）

**输入**：
- 晶体结构文件（POSCAR/CIF/XYZ/PDB）
- 模型选择（可多选）

**处理流程**：
1. 解析结构文件 → 提取元素、晶格、原子坐标
2. 构建原子图（原子作为节点，化学键作为边）
3. 调用ALIGNN模型推理
4. 返回预测结果

**输出**：
```json
{
  "structure_info": {
    "formula": "LiFe0.94Ti0.06PO4",
    "n_atoms": 28,
    "elements": {"Li": 4, "Fe": 3.76, "Ti": 0.24, "P": 4, "O": 16}
  },
  "predictions": {
    "jv_formation_energy_peratom_alignn": {
      "value": -2.532,
      "unit": "eV/atom",
      "source": "JARVIS-DFT"
    },
    "jv_optb88vdw_bandgap_alignn": {
      "value": 3.694,
      "unit": "eV",
      "source": "Materials Project"
    }
  }
}
```

**响应时间**：< 5秒（GPU）/ < 30秒（CPU）

---

#### F2: 批量预测（P0）

**输入**：
- 多个结构文件（最多100个/批）
- 模型选择

**处理流程**：
1. 文件校验（格式、完整性）
2. 批量提交到任务队列
3. 异步处理，进度展示
4. 完成后汇总结果

**输出**：
- 任务ID（用于查询）
- 进度百分比
- 完成后返回所有结构预测结果

**特殊处理**：
- 自动跳过格式错误的文件
- 生成汇总CSV/Excel报告

---

#### F3: 自定义掺杂生成（P1）

**输入**：
- 宿主结构文件（如LiFePO4的POSCAR）
- 掺杂元素列表（如Ti, V, Mn, Co）
- 浓度梯度（如1%, 2%, 5%, 10%）
- 掺杂位点（如Fe位、Li位、P位）

**处理**：
- 基于现有脚本 `generate_lfp_dopants_v4.py` 扩展
- 用户可配置生成规则
- 保存生成记录

**输出**：
- 生成的所有构型文件（POSCAR格式）
- 批量提交预测

---

#### F4: 多数据库对比（P1）

**输入**：单一结构文件

**处理**：
- 同时调用JARVIS模型和MP模型
- 计算两个数据库预测的相关性

**输出**：
```json
{
  "jarvis_prediction": {"bandgap": 0.058, "formation_energy": -2.070},
  "mp_prediction": {"bandgap": 3.694, "formation_energy": -2.532},
  "correlation_analysis": {
    "bandgap_reliability": "low（r=0.074）",
    "formation_energy_reliability": "high（r=0.94）"
  },
  "recommendation": "带隙参考MP，形成能参考两数据库均值"
}
```

---

#### F5: 3D晶体可视化（P1）

**功能**：
- 基于3Dmol.js实现Web端3D渲染
- 支持鼠标旋转、缩放、平移
- 高亮掺杂元素位置
- 显示晶格参数和原子标签

**格式支持**：
- 自动识别：POSCAR, CIF, XYZ, PDB

---

#### F6: 实验数据对比（P2）

**输入**：
- 预测结果
- 用户提供的实验值（手动输入或CSV导入）

**处理**：
- 计算MAE、RMSE、R²等指标
- 可视化误差分布

**输出**：
- 误差统计表
- 预测vs实验散点图

---

#### F7: 文献数据关联（P2）

**功能**：
- 自动关联相似材料体系的文献数据
- 显示文献中的实验/计算值

**数据源**：
- 内置文献摘要数据库
- 或链接到外部数据库（Materials Project, JARVIS）

---

#### F8: 批量数据管理（P2）

**功能**：
- 预测历史记录
- 批量任务管理
- 结果导出（CSV/Excel/JSON）

**存储**：
- 本地文件存储
- 可配置数据保留期限

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| 单结构预测响应 | < 5秒（GPU）/ < 30秒（CPU） | 从上传到结果展示 |
| 批量任务排队 | < 10秒 | 任务提交到开始处理 |
| Web页面加载 | < 3秒 | 首屏完整渲染 |
| 3D可视化渲染 | < 2秒 | 结构加载完成 |

### 4.2 可用性要求

| 场景 | 要求 |
|-----|------|
| GPU不可用 | 自动降级到CPU模式，显示提醒 |
| 模型加载失败 | 显示具体错误，提供重试按钮 |
| 文件格式错误 | 明确提示错误原因（精确到行） |
| 网络中断 | 自动重连，进度保存 |

### 4.3 数据管理

| 需求 | 说明 |
|-----|------|
| 文件格式 | POSCAR, VASP, CIF, XYZ, PDB |
| 批量大小 | 单次最多100个文件 |
| 存储策略 | 本地文件存储，可配置路径 |
| 导出格式 | CSV, Excel, JSON |
